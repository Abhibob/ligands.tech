# PoseBusters Wrapper Skill

## Purpose
Run plausibility checks on one or more poses.

## Use this skill when
- a pose was generated by Boltz or gnina
- the user wants a sanity check before trusting scores
- you want to filter a shortlist of poses

## Config choice
- `mol`: ligand-only plausibility
- `dock`: ligand + protein
- `redock`: ligand + protein + reference ligand

## Command pattern
```bash
bind-posebusters check \
  --request examples/requests/posebusters_check.yaml \
  --json-out /tmp/pb.json
```

## Interpretation policy
- fatal failures usually mean low confidence
- a pass means "not obviously implausible by these checks", not "verified binder"

## Downstream use
Use the categorized failure lists to decide whether gnina or PLIP is worth running on the pose.
